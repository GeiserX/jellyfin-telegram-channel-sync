# CLAUDE.md — Jellyfin Telegram Channel Sync

## Overview
A daemon that syncs Jellyfin user access with Telegram channel membership. Automatically disables Jellyfin accounts when members leave the Telegram channel and re-enables them when they rejoin.

## Tech Stack
- Python 3.13
- Telethon (Telegram client)
- Requests (Jellyfin API)
- SQLite (persistent state)
- Docker (drumsergio/jellytelegram-sync)
- GitHub Actions CI

## Development
```bash
# Install dependencies
pip install -r app/requirements.txt

# Run locally
python app/sync.py

# Run tests
cd tests && pytest

# Docker
docker compose up
```

## Architecture
- `app/sync.py` — main sync loop logic
- `app/requirements.txt` — Python dependencies
- `tests/` — test suite
- `Dockerfile` — container build (python:3.13-slim base)
- `docker-compose.yml` — example deployment with volume mount for data persistence

## Key Rules
- Never hardcode Telegram API credentials or Jellyfin API keys; use environment variables
- Docker images use semver tags on Docker Hub (drumsergio/jellytelegram-sync)
- Threshold guardrail: skip sync if fetched members drop below configurable threshold (prevents mass-disable on API hiccups)
- SQLite database and Telegram session file must persist via bind-mounted volume
